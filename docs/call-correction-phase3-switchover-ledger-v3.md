## Acceptance Gates (G0-G5)

- **G0 (Config Safety):** `resolver_mode=shadow` by default in all production configs; `primary` only via explicit change.
- **G1 (Correctness Drift):** During primary trials, disagreement trend (`DW/SP/UC`) must stay within pre-approved runbook bounds for the trial window.
- **G2 (Resource Safety):** Resolver drop/pressure counters (`Q/K/C/R`, pressure/evict) must remain within runbook guardrails; no sustained queue depth escalation.
- **G3 (Pipeline Health):** No new ingest blocking symptoms, telnet instability, or sustained stabilizer backlog growth attributable to primary mode.
- **G4 (Determinism/Repro):** Same-date replay reruns with fixed config remain deterministic before and after config changes intended for cutover.
- **G5 (Main/Replay Parity):** Replay helper artifacts must be generated by the same policy logic as live cluster correction/stabilizer decisions, with zero unexplained decision-reason drift for fixed-date comparisons.

## Rollback Triggers and Actions

- **Trigger R1:** sustained disagreement breach (especially `DW`) beyond agreed thresholds.
- **Trigger R2:** resolver capacity/drop breach indicating pressure that risks hot-path safety.
- **Trigger R3:** operator-observed correction regressions impacting expected output quality.
- **Trigger R4:** any observed ingest/fan-out instability temporally correlated with primary mode.
- **Trigger R5:** main-vs-replay parity drift after a phase change where drift cannot be explained by intentional config differences.

**Immediate action for any trigger:** set `call_correction.resolver_mode` back to `shadow`, restart process, and collect incident slice evidence for TSR follow-up.

## Current Operational Next Step
